## 📋 更新概述

6. `game` - 手機遊戲 / 網頁遊戲
7. `other` - 其他（使用預設提示）

## 📊 提示內容結構

每個專案類型都包含以下提示內容：

```typescript
interface ProjectTypeHints {
  usageScenario: {
    placeholder: string;      // 輸入框 placeholder
    examples: string[];       // 範例情境列表
    hint: string;             // 小提示文字
  };
  goals: {
    commonGoals: string[];    // 常見目標選項
    placeholder: string;      // 輸入框 placeholder
    hint: string;             // 小提示文字
  };
  features: {
    placeholder: string;      // 輸入框 placeholder
    examples: string[];       // 功能範例列表
    hint: string;             // 小提示文字
  };
  outputs: {
    examples: string[];       // 產出範例
    hint: string;             // 小提示文字
  };
  reference: {
    hint: string;             // 小提示文字
  };
  integrations: {
    commonIntegrations: string[]; // 常見整合選項（目前未使用）
    hint: string;             // 小提示文字
  };
  budgetSchedule: {
    hint: string;             // 小提示文字
  };
  deliverables: {
    examples: string[];       // 交付物範例（目前未使用）
    hint: string;             // 小提示文字
  };
  additional: {
    hint: string;             // 小提示文字
  };
}
```

## 📝 各類型詳細提示內容

### 1. 官方形象網站 (website)

**Step 2 - 使用情境**:
- 範例：訪客進入首頁 → 瀏覽關於我們 → 查看服務項目 → 填寫聯絡表單
- 提示：描述訪客會如何瀏覽和使用您的網站，包括主要頁面流程

**Step 3 - 目標**:
- 常見目標：提升品牌形象與專業度、增加線上曝光、讓客戶更容易找到我們、展示公司服務與案例
- 提示：網站的主要目的是建立形象還是獲取客戶？這會影響設計和功能重點

**Step 4 - 功能**:
- 範例：公司介紹/關於我們、服務項目展示、案例作品集、聯絡表單、最新消息/部落格、多語言切換、響應式設計、SEO 優化
- 提示：官方網站通常需要：公司介紹、服務展示、聯絡方式、案例作品等基本頁面

**Step 7 - 整合**:
- 提示：官方網站常需要整合 Google Analytics、地圖、社群媒體等服務

**Step 8 - 預算時程**:
- 提示：官方網站通常需要 1-3 個月開發時間，預算範圍較廣，建議提供明確範圍

### 2. 電商平台 (ecommerce)

**Step 2 - 使用情境**:
- 範例：顧客搜尋商品 → 查看商品詳情 → 選擇規格 → 加入購物車 → 結帳付款
- 提示：描述完整的購物流程，從瀏覽商品到完成訂單的每個步驟

**Step 3 - 目標**:
- 常見目標：增加線上銷售額、提升購物轉換率、擴大銷售通路、自動化訂單處理
- 提示：電商平台的核心目標是銷售轉換，思考如何提升購物體驗和轉換率

**Step 4 - 功能**:
- 範例：商品分類與搜尋、商品詳情頁、購物車功能、會員註冊登入、多種付款方式、訂單查詢與追蹤、優惠券/折扣碼、庫存管理、後台訂單管理、商品評價系統
- 提示：電商平台需要完整的購物流程：商品展示、購物車、付款、訂單管理

**Step 7 - 整合**:
- 提示：電商平台必須串接金流和物流，這是核心功能，務必確認

**Step 8 - 預算時程**:
- 提示：電商平台開發時間通常需要 2-4 個月，預算範圍較大，建議提供明確範圍

---

### 3. ERP / CRM 系統 (erp_crm)

**Step 2 - 使用情境**:
- 範例：員工登入 → 填寫請假單 → 主管審核 → 系統記錄 → 通知 HR
- 提示：描述內部員工如何使用系統處理業務流程，包括審核、統計等環節

**Step 3 - 目標**:
- 常見目標：減少重複手動作業、自動化業務流程、提升資料準確性、即時掌握營運數據
- 提示：ERP/CRM 系統的核心是流程自動化和數據分析，思考哪些流程需要優化

**Step 4 - 功能**:
- 範例：客戶資料管理（CRM）、訂單/報價管理、庫存管理、財務會計模組、人事管理、報表分析與統計、權限管理、審核流程、資料匯入匯出、通知提醒系統
- 提示：ERP/CRM 系統需要多個功能模組，建議先列出最核心的模組需求

**Step 7 - 整合**:
- 提示：ERP/CRM 系統常需要與現有系統整合，確認需要串接哪些外部系統

**Step 8 - 預算時程**:
- 提示：ERP/CRM 系統開發時間通常需要 3-6 個月以上，預算範圍較大，建議分階段開發

### 4. Line LIFF / Line 工具 (chatbot)

**Step 2 - 使用情境**:
- 範例：用戶打開 Line → 點擊選單 → 進入 LIFF 頁面 → 填寫表單 → 提交資料 → 收到確認訊息
- 提示：描述用戶在 Line 中如何使用您的服務，包括選單操作和頁面流程

**Step 3 - 目標**:
- 常見目標：透過 Line 接觸更多客戶、提供 Line 內一站式服務、自動化客服與行銷、整合 Line 會員系統
- 提示：Line 工具的核心是利用 Line 的用戶基礎和生態圈，思考如何提升互動和轉換

**Step 4 - 功能**:
- 範例：Line Login 登入、Line 選單（Rich Menu）、LIFF 網頁應用、訊息推播（Broadcast）、好友分群標籤、圖文選單、Quick Reply、Flex Message、Webhook 事件處理、會員資料管理
- 提示：Line 工具需要整合 Line Messaging API、LIFF、Login 等功能

**Step 7 - 整合**:
- 提示：Line 工具必須串接 Line 官方 API，確認需要哪些 Line 服務

**Step 8 - 預算時程**:
- 提示：Line 工具開發時間通常需要 1-3 個月，預算範圍中等，需考慮 Line 官方帳號費用

- 範例：用戶下載 App → 註冊會員 → 瀏覽內容 → 完成預約 → 收到確認通知
- 提示：描述用戶在手機上如何使用 App，包括主要功能和操作流程

**Step 3 - 目標**:

**Step 4 - 功能**:
- 範例：會員註冊登入、推播通知、相機拍照/上傳、GPS 定位、離線功能、QR Code 掃描、生物辨識、社群分享、內建聊天功能、支付功能

- 提示：遊戲常需要串接付費、登入、排行榜、廣告等服務

**Step 8 - 預算時程**:
- 提示：遊戲開發時間通常需要 3-6 個月以上，預算範圍很大，建議分階段開發

## 🔄 運作機制

1. **使用者選擇專案類型**（Step 1）
   - 選擇「全新開發」後，選擇具體類型（website、ecommerce 等）

2. **後續步驟自動載入對應提示**
   - 每個步驟組件呼叫 `getProjectTypeHints(data.projectType)`
   - 取得該類型的提示內容配置

3. **顯示針對性內容**
   - Placeholder 文字
   - 範例選項
   - 提示說明

4. **未選擇類型時使用預設**
   - 如果 `projectType` 為 `undefined` 或 `'other'`
   - 使用 `DEFAULT_HINTS` 通用提示
